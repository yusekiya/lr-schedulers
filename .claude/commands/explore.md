# Issue Analysis Command

A command to analyze given issues and identify the root causes of problems.

## Instructions

$ARGUMENTS

## Execution Steps

Follow this systematic approach to understand the current state and analyze issues:

1. Understand the Project
    - Understand the project's directory structure
    - Read `docs/architecture.md` to understand the software architecture
    - Read `docs/project-requirements.md` to understand project requirements

2. Analyze the Issue
    - Understand the issue to be resolved from the instructions
    - Investigate files related to this issue from the codebase
    - Find the problematic code in the related files
    - Analyze the code to identify the root cause of the problem
    - Check for related issues or similar problem patterns

## Output

Include the following in the output:

- **Summary**
    - A concise summary of the issue and its cause
- **Root Cause**
    - Details of the root cause of the problem
- **Related Information**
    - Investigation results on related issues or similar problems
- **Next Steps**
    - `/plan`: Design a plan

## Important Notes

- Do NOT add or modify any code during this investigation phase
