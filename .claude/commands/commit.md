# Commit Command

Commit the changes.

## Instructions

$ARGUMENTS

## Execution Steps

Follow this systematic approach:

1. Understand the Context
    - Understand the project's directory structure
    - Read `docs/git-guidelines.md` to understand the project's Git conventions
    - Understand the changes with the `git diff` command
    - Understand the reason for changes from previous output results
    - Understand the instructions (if any)

2. Commit
    - Check the commit command in the list of development commands (`docs/common-commands.md`)
    - Commit following the project's Git conventions

3. Push
    - Show the user the list of commits and request permission to push
    - Check the push command in the list of development commands (`docs/common-commands.md`)
    - If user permission is granted, push following the project's Git conventions

## Output

Include the following in the output:

- **Commit List**
    - Display the created commits in chronological order
    - Display commit messages
- **Next Steps**
    - `/pr`: Create a pull request

## Important Notes

- Always use English for commit titles and descriptions
- Do NOT add or modify any code
